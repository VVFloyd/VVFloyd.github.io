做个小树洞02（接入图片服务，HttpURLConnection自定义http post请求）

SpringBoot java
练习项目
发布日期:   2020-10-15

今天着手做个基础功能：写博客。我们先不纠结前端，postman就是最好的页面(●ˇ∀ˇ●)。
大致有这么几个需求：

  1. 用户可以插入新文章；
  2. 插入文章可以选择可见对象（级别：全部可见、粉丝可见、好友可见、黑名单不可见）；
  3. 未提交的部分保存在草稿箱；
  4. 用户可以给文章设置TAG，未设置将从文章标题提取关键字；
  5. 封面图片服务；

接入图片服务

使用SM.MS的免费服务，文档地址：https://doc.sm.ms/#api-User-Get_Token

参考文档，找出我们需要的接口，即上传接口（实际使用时还可能用得到删除接口，但是我们只是举个栗子，所以先实现上传接口就好了）。
请参考Upload - Upload Image这个接口，关注它的请求类型和入参（Authorization可以不传），在postman中试着调一下，成功~

那么就是说只要取返回值里的url字段，就能拿到我们需要的值了。但是这里要注意，如果同样的请求重复一次，返回值结构变化了，后续代码中要注意处理这个问题。
避免用户上传图片后删除，之后又上传的情况。

要请求外部接口，首先需要一个类帮助我们处理连接和数据传输。这里叫HttpUtil.这个工具类网上一搜有很多，笔者发现了一个用java8写的，同时内容精简可靠的工具类。
拿来研究一下，特别感谢作者“纠结的猫”源地址：https://www.cnblogs.com/paranoidCAT/p/13695276.html
这个工具类只有两个共有方法，一个get一个post，于是我们从这里入手就好。


同时可以关注下Collections的emptyList() emptyMap() emptySet()方法。这三个方法主要作用有以下两点，这里用到的就是第二点（引用自：https://blog.csdn.net/zwt0909/article/details/52426787）：

  • 方法内部会返回static final成员，创建后相当于常量可重复引用，当需要使用一个空集合时不用new去分配内存，比如一个测试用例API接口就需要一个Map<String,Object>,若此时只需要一个空map跑用例，直接用Collections.emptyMap()作为参数即可

  • 防止空指针出现，当你的代码需要一个集合而这个集合可能不存在，此时尽量使用空集合而不是null，因为集合一个常用的操作就是遍历，你不知道你返回的结果在后续会不会被遍历。比如一个查询步骤返回一个集合，当返回一个空集合是就可以用这类方法，还可以防止后续对这个空集合再做add操作

但是我们发送请求的时候还需要特殊处理，因为发送的请求中有图片，也有String类型的参数。找了找没有成熟的工具类，于是只能自己根据标准请求报文拼一个。

–balbalaaaa这个是数据分割线，具体是什么没有要求，可以使用时间戳。请求报文参数以数据分割线开头，并以其结束。参数的key放在name里，空一行后写value。
直接描述可能有点抽象，上述例子中提交了三个参数：{“lng”:”116.361545”,”lat”:”39.979006”,”images”:”图片数据”},根据这个来仔细看看，就能看懂了哦。
注意报文格式是固定的，所以空行不能省略，也不能乱加。
如下是改造后的方法，注释已经写在上面啦：

 //数据分割线
    private static final String BOUNDARY = String.valueOf(System.currentTimeMillis());

public static String post(String urlStr, Map<String, String> params,
                                   Map<String, File> images) {
        try {
            URL url = new URL(urlStr);
            HttpURLConnection con = (HttpURLConnection) url.openConnection();

            //一些post请求的设置
            con.setConnectTimeout(5000);
            con.setDoInput(true);
            con.setDoOutput(true);
            con.setUseCaches(false);
            con.setRequestMethod("POST");
            con.setRequestProperty("Connection", "Keep-Alive");
            con.setRequestProperty("Charset", "UTF-8");
            //不添加这个设置会报403，包含了操作系统、安全等级、浏览器语言等设置
            con.setRequestProperty("User-Agent",
                    "Mozilla/5.0 (Windows; U; Windows NT 6.1; zh-CN; rv:1.9.2.6)");
            con.setRequestProperty("Content-Type", "multipart/form-data; boundary="+ BOUNDARY);

            DataOutputStream dos = new DataOutputStream(con.getOutputStream());;

            //换行符
            final String newLine = "\r\n";
            //请求报文特定字符
            final String boundaryPrefix = "--";

            //添加请求参数
            for(Map.Entry<String, String> s : params.entrySet()){
                StringBuilder sb = new StringBuilder();
                sb.append(newLine);
                sb.append(newLine);
                sb.append(boundaryPrefix).append(BOUNDARY).append("\r\n")
                        .append("Content-Disposition: form-data;name=\""+s.getKey()+"\"")
                        .append(newLine).append("Content-Type: text/plain; charset=UTF-8")
                        .append(newLine)
                        .append("Content-Transfer-Encoding: 8bit")
                        .append(newLine)
                        .append(newLine);

                dos.write(sb.toString().getBytes());
                dos.write(s.getValue().getBytes());
            }

            //如果请求中包含图片
            if (images != null) {
                StringBuffer strBf = new StringBuffer();
                strBf.append(newLine)
                        .append(boundaryPrefix).append(BOUNDARY).append("\r\n")
                        .append("Content-Disposition: form-data;name=\"smfile\";filename=\"" + images.get("smfile").getName() + "\"\r\n")
                        .append("Content-Type:application/octet-stream\r\n")
                        .append("Content-Transfer-Encoding: binary")
                        .append(newLine)
                        .append(newLine);

                dos.write(strBf.toString().getBytes());

                DataInputStream in = new DataInputStream(new FileInputStream(
                        images.get("smfile")));
                byte[] bufferOut = new byte[1024];
                int bytes = 0;
                // 每次读1KB数据,并且将文件数据写入到输出流中
                while ((bytes = in.read(bufferOut)) != -1) {
                    dos.write(bufferOut, 0, bytes);
                }
                in.close();
            }

            // 报文结尾
            // 最后添加换行
            dos.write(newLine.getBytes());
            // 定义最后数据分隔线，即--加上BOUNDARY再加上--。
            byte[] end_data = (newLine + boundaryPrefix + BOUNDARY + boundaryPrefix + newLine)
                    .getBytes();
            // 写上结尾标识
            dos.write(end_data);
            dos.flush();
            dos.close();

            //读取结果
            BufferedReader reader = new BufferedReader(new InputStreamReader(
                    con.getInputStream()));
            String result="";
            String line = null;
            while ((line = reader.readLine()) != null) {
                System.out.println(line);
                result += line;
            }
            return result;
        } catch (MalformedURLException e) {
            e.printStackTrace();
        } catch (ProtocolException e) {
            e.printStackTrace();
        } catch (IOException e) {
            e.printStackTrace();
        }
        return "error";
    }

service如下：

public Map upload(MultipartFile picture) throws Exception {

        Map<String,String> params=new HashMap<>();
        params.put("format","json");
        Map<String, File> image=new HashMap<>();
        image.put("smfile", MultipartFileToFile.multipartFileToFile(picture));

        String result = HttpUtil.post("https://sm.ms/api/v2/upload", params, image);
        JSONObject jsonObject = JSON.parseObject(result);
        String url="";
        if("true".equals(jsonObject.getString("success"))){
            JSONObject data = JSON.parseObject(jsonObject.getString("data"));
            url=data.getString("url");
        }else if("image_repeated".equals(jsonObject.getString("code"))){
            url=jsonObject.getString("images");
        }

        Map<String,String> rt = new HashMap<>();
        rt.put("url",url);

        return rt;
    }

这样就可以拿到图片上传后的地址了。不过还有很多可以优化的地方：

  1. 返回值可以封装；
  2. 异常处理；
  3. 全局日志；
  4. 这个接口请求肉眼可见的很慢，所以对重复请求可以放到redis里处理；
  5. 这个方法里有些内容可以用java8优化。

先处理第五点，优化后代码如下（但是感觉还不够优美，待后续打磨）：

url = Optional.ofNullable(JSON.parseObject(result))
            .filter(a ->  Objects.equals(a.getString("success"), "true"))
            .map(a -> (JSONObject)a.get("data"))
            .map(a -> a.getString("url"))
            .orElse(Optional.ofNullable(JSON.parseObject(result))
                    .filter(a -> "image_repeated".equals(a.getString("code")))
                    .map(a -> a.getString("images"))
                    .toString());

其他优化点在后续会一一补全。在这篇先简单以postman请求结果结尾。
