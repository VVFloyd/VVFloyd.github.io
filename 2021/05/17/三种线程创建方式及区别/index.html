三种线程创建方式及区别

java java基础
java
发布日期:   2021-05-17

起因：昨天刷B站的时候看到一个推荐视频，内容是月薪一万六面试实录。所及内容多为基础，但是其中有个问题却难到了我，问的是线程创建的三种方式。
平时工作中多以 executorService.execute(() -> {}); 的方式，利用线程池创建，于基础却忘得一干二净。于是今日来补一补。

1.Thread,Runnable

首先來看Runnable,基础用法如下:

public class RunnableTest {
    public static void main(String[] args) {
        new Thread(new RunnableDemo()).start();
    }

    public static class RunnableDemo implements Runnable{
        @Override
        public void run() {
            System.out.println("hahaha~");
        }
    }
}

再來看Thread,基础用法如下：

public class ThreadTest {
    public static void main(String[] args) {
        new ThreadDemo().start();
    }

    public static class ThreadDemo extends Thread{
        @Override
        public void run(){
            System.out.println("hahaha~");
        }
    }
}

可见，这两种方式是十分相似的，唯一的区别是runnable需要调用thread的构造方法：
public Thread(Runnable target) {
init(null, target, “Thread-“ + nextThreadNum(), 0);
}
来创建thread.进而调用他的start方法。

那runnable和thread真正的区别是什么呢？
答：本质上没有区别。因为thread其实是runnable的实现类，前者封装了后者。单独使用runnable，依旧要传入thread的构造函数来调用。

我们来看thread的源码

@Override
    public void run() {
        if (target != null) {
            target.run();
        }
    }

可见thread其实是调用了target的run方法，而target正是runnable.

/* What will be run. */
private Runnable target;

在单独使用tread时，往往重写了run方法，才导致我们看不到runnable的影子。

有一种论调是“Runnable更容易可以实现多个线程间的资源共享，而Thread不可以”，但是这种观点是错误的。参考链接：https://blog.csdn.net/zhaojianting/article/details/97664370

改造一下我们的栗子也可以证明，这不过是由于是否是同一个对象引起的问题。

    public static void main(String[] args) {
        new Thread(new RunnableDemo()).start();
        new Thread(new RunnableDemo()).start();

        RunnableDemo runnableDemo = new RunnableDemo();
        new Thread(runnableDemo).start();
        new Thread(runnableDemo).start();

    }

    public static class RunnableDemo implements Runnable{
        private int ticket = 5;
        @Override
        public void run() {
            System.out.println("hahaha~"+ticket--);
        }
    }

2. Callable

再来看看Callable.

public class CallableTest {
    public static void main(String[] args) throws ExecutionException, InterruptedException {
        FutureTask<String> stringFutureTask = new FutureTask<>(new CallableDemo());
        new Thread(stringFutureTask).start();
        //获取结果
        System.out.println(stringFutureTask.get());
    }

    public static class CallableDemo implements Callable<String> {

        @Override
        public String call() throws Exception {
            return "hahaha~";
        }
    }
}

可见，callable同样需要thread包装，FutureTask继承了runnable.callable可以获取结果。
因为callable的返回值是在未来执行返回的，所以别忘了给get方法设置一个超时时间。

//源码
public V get(long timeout, TimeUnit unit)
        throws InterruptedException, ExecutionException, TimeoutException {
        if (unit == null)
            throw new NullPointerException();
        int s = state;
        if (s <= COMPLETING &&
            (s = awaitDone(true, unit.toNanos(timeout))) <= COMPLETING)
            throw new TimeoutException();
        return report(s);
    }

callable使用的常见不多，往往要求在主线程中需要执行比较耗时的操作时，但又不想阻塞主线程时，可以把这些作业交给Future对象在后台完成，当主线程将来需要时，就可以通过Future对象获得后台作业的计算结果或者执行状态。
